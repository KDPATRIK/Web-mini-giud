01 — Основы

Структура веб-разработки

Веб-разработка делится на четыре слоя, каждый из которых отвечает за свою часть работы.

Frontend

Клиентская часть

Всё, что видит пользователь: HTML, CSS, JavaScript. Фреймворки — React, Vue. Инструменты сборки — Vite, Webpack.

Backend

Серверная часть

Логика приложения, обработка данных, API. Языки: Node.js, Python, Go. Взаимодействует с фронтендом через REST API.

Database

База данных

Хранилище данных. SQL (PostgreSQL, MySQL) — для структурированных данных. NoSQL (MongoDB) — для гибких.

DevOps

Инфраструктура

Docker, CI/CD, облачные провайдеры (AWS, GCP). Отвечает за деплой, масштабирование и надёжность.

Protocols

Протоколы

HTTP/S, WebSocket, DNS, SSL/TLS, CDN. Связывают все слои системы между собой.

Security

Безопасность

Auth / JWT, CORS, шифрование. Защищает данные пользователей и обеспечивает авторизацию.

HTML Разметка
Задаёт структуру страницы — заголовки, параграфы, кнопки, формы, ссылки.
CSS Стили
Отвечает за внешний вид: цвета, шрифты, отступы, анимации, адаптивность.
JavaScript Логика
Делает страницу интерактивной: обработка кликов, запросы к серверу, динамический контент.
React / Vue Фреймворки
Помогают строить сложные интерфейсы из переиспользуемых компонентов.
02 — HTML

Что такое HTML

HTML — язык разметки, на котором написана структура каждой веб-страницы. Браузер читает HTML и превращает его в визуальную страницу.

DOM-дерево страницы

<html>
<head>
<title> — заголовок вкладки
<meta> — SEO, кодировка
<body>
<header> — шапка
<main> — основной контент
<h1>–<h6> заголовки
<p> параграфы
<img> <a> <div>
<footer> — подвал

Пример разметки

index.html 
<!-- Структура HTML-страницы -->
<html lang="ru">
  <head>
    <title>Мой сайт</title>
    <meta charset="UTF-8">
  </head>

  <body>
    <header>
      <h1>Заголовок сайта</h1>
    </header>

    <main>
      <h2>О нас</h2>
      <p>Текст параграфа...</p>
      <img src="photo.jpg"
           alt="Фото">
      <a href="/contact">
        Связаться</a>
    </main>

    <footer>
      <p>© 2025</p>
    </footer>
  </body>
</html>
Семантика

Теги <header>, <main>, <footer> говорят браузеру о смысле блока. Улучшают SEO и доступность.

Атрибуты

Параметры тегов: src, href, alt, class, id. Дают дополнительную информацию элементу.

04 — Архитектура

Виды структуры сайта

Три основных способа организовать связи между страницами. У каждого — своя область применения.

Линейная

Страницы идут одна за другой, как шаги. Пользователь движется по заданному маршруту без ответвлений.

→ Лендинг, квиз, онбординг

Иерархическая

Дерево: главная → разделы → подстраницы. Самый распространённый вид, подходит для большинства сайтов.

→ Корпоративный сайт, магазин

Сетевая (паутина)

Страницы связаны между собой произвольно. Можно попасть с любой страницы на любую через перекрёстные ссылки.

→ Википедия, новостной сайт

НА ПРАКТИКЕ

Большинство сайтов используют иерархию как основу и добавляют перекрёстные ссылки там, где это необходимо — например, «Похожие статьи» в блоге или «С этим товаром покупают» в магазине.
